### 目标用户 & 使用场景

#### Q1: 这个便签应用是**纯个人使用**，还是未来可能支持**多用户/团队协作**？

**A**: 纯个人使用

**A**: 需要这个选项，可以提供一个默认的开机自启动，用户可以取消

### 核心功能细化

#### Q3: 便签编辑器使用**纯文本**还是**富文本**？

**A**: 先纯文本，后续升级为富文本

#### Q4: 便签窗口外观偏好哪种？

**A**: c) 自定义标题栏 — 去掉系统标题栏，自己画一个更好看的

#### Q5: 便签是否有**最大数量限制**？

**A**: 无上限

#### Q6: 删除便签的方式？

**A**: b) 回收站 — 删除后进回收站，可以恢复

#### Q7: 每个便签窗口上需要哪些**快捷操作按钮**？

**A**: c) 两者都要，用户可切换卡片网格/列表视图

## 澄清结论

经过澄清，需求理解如下：

1. **定位**：纯个人使用的轻量级桌面便签工具，当前全部本地运行，架构预留远程 API 对接能力
2. **窗口设计**：采用自定义标题栏（frameless + 自绘），现代扁平设计风格（类 Windows 11 Sticky Notes）
3. **编辑器**：MVP 阶段使用纯文本，架构预留富文本升级能力
4. **数据管理**：JSON 文件存储，支持回收站恢复，支持数据导入/导出
5. **便签窗口**：默认 300×300 可自定义大小，包含新建/删除/颜色/置顶/关闭 5 个快捷按钮
6. **管理面板**：支持卡片网格 + 列表两种视图切换
7. **系统集成**：开机自启动（可配置）、系统托盘常驻、全局快捷键
8. **平台**：Windows + macOS 双平台支持
9. **发布**：需要自动更新功能
