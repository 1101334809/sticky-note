## 需求背景

Image Toolkit 是一款面向**普通用户**的桌面端图片处理工具，包含 SVG 查看、图片压缩、格式转换三大核心模块。当前版本在操作流程、交互体验、视觉适配方面存在多处不足，用户在批量操作（上百个文件）场景下体验尤为欠佳。

本次迭代目标：**提升操作人性化程度，让用户拥有更流畅、更直觉的使用体验。**

## 目标用户

| 用户角色     | 特征                               | 使用场景                             |
| ------------ | ---------------------------------- | ------------------------------------ |
| 普通用户     | 非技术背景，需要简洁直观的操作界面 | 日常图片压缩、格式转换、SVG 图标管理 |
| 批量操作用户 | 单次处理上百个文件                 | 批量压缩、批量转换、批量导出         |

## 功能描述

本次迭代涵盖 **6 大优化方向、30+ 优化项**，三批全部纳入实施范围。

### 功能边界

- ✅ 包含：30+ 优化项三批全部纳入本次迭代

## 用户故事

- 作为**普通用户**，我希望**添加文件后能单独移除某个文件**，以便**不需要重新选择所有文件**
- 作为**普通用户**，我希望**拖入文件时自动切换到对应模块**，以便**不需要手动切换页面**
- 作为**普通用户**，我希望**点击 SVG 能看到大图预览**，以便**确认图标内容后再操作**
- 作为**普通用户**，我希望**压缩完成后能一键打开输出文件夹**，以便**快速找到处理结果**
- 作为**普通用户**，我希望**格式转换默认保留原文件**，以便**不用担心原图丢失**
- 作为**普通用户**，我希望**Ctrl+Z 撤销误操作**，以便**快速恢复到操作前的状态**
- 作为**普通用户**，我希望**在浅色模式下也能正常使用**，以便**在不同环境下都有良好体验**

## 业务规则

| 编号 | 规则                 | 说明                                                |
| ---- | -------------------- | --------------------------------------------------- |
| BR01 | 文件去重基于路径     | 同一路径文件不可重复添加，提示"文件已存在"          |
| BR02 | 追加模式仅限压缩模块 | 格式转换模块无此问题，保持现有行为                  |
| BR03 | 混合拖拽自动拆分     | 拖入 SVG + 图片混合文件时，按格式自动分发到对应模块 |
| BR04 | 输出目录路径记忆     | 选择的输出目录持久化，下次启动保留上次路径          |
| BR05 | 保留原文件默认开启   | 格式转换"保留原文件"开关默认打开                    |
| BR06 | 撤销栈完整多级       | 可撤销文件列表变更 + 颜色修改等参数变更             |
| BR07 | SVG 导出 PNG 双模式  | 支持倍率（1x/2x/3x）和自定义像素两种方式            |
| BR08 | 浅色模式全局适配     | 所有组件统一使用 CSS 变量 / Naive UI 主题 token     |

## 边界与异常

| 场景             | 触发条件                 | 处理方式                                         |
| ---------------- | ------------------------ | ------------------------------------------------ |
| 重复文件拖入     | 文件路径与已有文件相同   | 跳过并 Toast 提示"文件已存在"                    |
| 混合格式拖入     | 拖入包含多种格式的文件组 | 按格式自动拆分，分发到对应模块                   |
| 压缩中取消       | 用户点击"取消"按钮       | 中断任务，已完成文件保留结果，未完成标记"已取消" |
| 压缩单文件失败   | 单个文件压缩出错         | 标记失败状态，显示重试按钮和错误原因             |
| 格式转换格式相同 | 源文件格式 = 目标格式    | 弹出确认提示，用户确认后继续                     |
| 撤销栈溢出       | 操作次数超过撤销栈上限   | 丢弃最早的操作记录                               |
| 输出目录不存在   | 记忆的路径已被删除       | 回退到默认（源文件同目录），提示路径已失效       |
| 上百文件加载     | 一次性添加大量文件       | 显示骨架屏/进度，使用虚拟列表保证性能            |

## 非功能需求

| 类型       | 要求                                                     |
| ---------- | -------------------------------------------------------- |
| **性能**   | 上百个文件的列表渲染流畅（虚拟列表），操作响应 < 200ms   |
| **兼容**   | 浅色/深色模式全局适配，Windows/macOS 系统文件选择器兼容  |
| **持久化** | 输出目录路径、用户偏好设置（音效开关等）持久化到本地存储 |
| **可用性** | 面向普通用户，界面语言简洁无技术术语，操作可撤销         |
