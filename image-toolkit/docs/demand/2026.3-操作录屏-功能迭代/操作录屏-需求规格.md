## 需求背景

Image Toolkit 是一款面向**普通用户**的桌面端工具库（Electron + Vue 3）。用户在日常工作中经常需要在**多个 Windows 应用之间**执行重复性操作流程，例如：

1. **跨应用批量处理**：从文件夹选取文件 → 拖入 Image Toolkit 压缩 → 将结果复制到另一个目录 → 在其他应用中使用
2. **固定化日常流程**：每天重复相同的打开应用 → 设置参数 → 执行操作 → 保存结果流程
3. **自动化枯燥操作**：在 Windows 桌面上重复执行大量点击、输入、拖拽等机械操作

本次迭代目标：**为 Image Toolkit 增加 Windows 系统级的操作录制与回放能力。录制用户在 Windows 任意应用/桌面上的鼠标和键盘操作，保存为可复用的操作脚本，支持一键回放自动执行，实现跨应用的工作流自动化。**

## 核心概念

| 术语         | 定义                                                                    |
| ------------ | ----------------------------------------------------------------------- |
| **操作脚本** | 一组按时间顺序排列的系统级操作事件（鼠标/键盘）的集合，以 JSON 文件存储 |
| **录制**     | 通过 Windows 全局钩子捕获用户在任意应用中的鼠标和键盘操作并保存的过程   |
| **回放**     | 读取操作脚本，通过 Windows API 模拟输入自动执行操作序列的过程           |
| **全局钩子** | Windows `SetWindowsHookEx` API，在系统级别拦截输入事件                  |

### 功能边界

- ✅ 包含：Windows 桌面/任意应用上的鼠标和键盘操作录制
- ✅ 包含：通过 Win32 API 模拟输入实现操作回放
- ✅ 包含：操作脚本的保存、管理、编辑、导入导出

- ❌ 不包含：macOS / Linux 平台支持（仅 Windows）

- ❌ 不包含：条件分支/循环逻辑等编程能力
- ❌ 不包含：录制视频画面（仅录制操作事件数据）

## 用户故事

- 作为**高频操作用户**，我希望**录制我在 Windows 桌面上的全部鼠标和键盘操作**，以便**下次一键回放自动执行相同流程**
- 作为**效率追求者**，我希望**用一个全局热键就能开始/停止录制**，以便**不用切回 Image Toolkit 窗口操作**
- 作为**普通用户**，我希望**回放前有倒计时让我准备好窗口**，以便**回放在正确的界面上执行**
- 作为**普通用户**，我希望**有紧急停止热键随时终止回放**，以便**出现意外时立即中断自动操作**
- 作为**团队协作用户**，我希望**将操作脚本导出分享给同事**，以便**团队使用统一的操作流程**
- 作为**批量处理用户**，我希望**设置脚本循环回放多次**，以便**自动重复执行相同操作**
- 作为**普通用户**，我希望**编辑脚本中的坐标**，以便**适配不同屏幕分辨率的电脑**
- 作为**普通用户**，我希望**设定点击频率后按一个键就能自动连点**，以便**不用手动反复点击鼠标**
- 作为**效率追求者**，我希望**设置多个坐标点轮流自动点击**，以便**自动化多位置的重复点击操作**
- 作为**普通用户**，我希望**保存连点器配置为预设**，以便**不同场景快速切换使用**

## 技术方案概述

| 技术点               | 方案                                                                         |
| -------------------- | ---------------------------------------------------------------------------- |
| **全局鼠标钩子**     | 通过 Node.js 原生 C++ 插件（N-API）调用 `SetWindowsHookEx(WH_MOUSE_LL, ...)` |
| **全局键盘钩子**     | 通过 Node.js 原生 C++ 插件调用 `SetWindowsHookEx(WH_KEYBOARD_LL, ...)`       |
| **输入模拟（回放）** | 通过 `SendInput` Win32 API 模拟鼠标移动/点击和键盘按键                       |
| **窗口信息获取**     | `GetForegroundWindow` + `GetWindowText` 获取当前前台窗口标题                 |
| **全局热键**         | `RegisterHotKey` API 或通过全局键盘钩子识别特定组合键                        |
| **屏幕截图**         | `BitBlt` API 或 Electron `desktopCapturer` 截取屏幕快照                      |
| **进程间通信**       | Electron IPC（渲染进程 ↔ 主进程）传递录制控制指令和事件数据                  |
| **原生模块构建**     | 使用 `node-gyp` 或 `cmake-js` 编译 C++ 原生插件，随 Electron 打包分发        |

## 业务规则

| 编号 | 规则                       | 说明                                                             |
| ---- | -------------------------- | ---------------------------------------------------------------- |
| BR01 | 坐标使用屏幕绝对坐标       | 鼠标事件使用基于屏幕左上角的绝对像素坐标（非窗口相对坐标）       |
| BR02 | 录制不阻断用户操作         | 钩子为只读监听，不拦截/修改用户的实际输入                        |
| BR03 | 鼠标移动采样节流           | 鼠标移动事件按 50ms 间隔采样或 >5px 位移时采样，避免数据量爆炸   |
| BR04 | 回放前倒计时 3 秒          | 给用户切换到目标窗口的准备时间                                   |
| BR05 | 紧急停止热键最高优先级     | `Esc` 立即终止回放和连点，不可被覆盖                             |
| BR06 | 记录屏幕分辨率             | 脚本文件记录录制时的屏幕分辨率，回放时若分辨率不同则弹出警告     |
| BR07 | 时间戳采用相对偏移         | 事件时间戳为录制开始时间的毫秒偏移量                             |
| BR08 | 回放期间录制热键不响应录制 | 防止回放过程中意外触发新的录制                                   |
| BR09 | 脚本默认存储位置           | 应用数据目录下 `scripts/` 文件夹                                 |
| BR10 | 回放期间显示悬浮控制窗     | 始终置顶的小窗口，显示进度和控制按钮                             |
| BR11 | 连点器默认跟随鼠标         | 连点器默认在当前鼠标位置点击，用户可切换为固定坐标模式           |
| BR12 | 连点器频率上限 50 CPS      | 防止频率过高导致系统卡顿或被目标应用视为异常                     |
| BR13 | 连点器热键不与录制冲突     | 连点器 `F6` 热键与录制 `Ctrl+Alt+R` / 回放 `Ctrl+Alt+P` 互不干扰 |
| BR14 | 连点器和回放互斥           | 连点器运行中不可启动回放，反之亦然                               |
| BR15 | 录制自动过滤控制热键       | `Ctrl+Alt+R` 等录制控制操作不记录到脚本中                        |
| BR16 | 敏感模式支持               | 录制中可临时暂停键盘捕获，保护密码等敏感输入                     |
| BR17 | 智能等待支持               | 回放中支持等待特定窗口标题出现后再继续，超时可选跳过/继续/停止   |

## 验收标准

### 第一批（高优先级 · 核心能力）

- [ ] AC01: 点击「开始录制」或全局热键后，系统级鼠标钩子成功安装，在任意应用中的鼠标点击被捕获
- [ ] AC02: 系统级键盘钩子成功安装，在任意应用中的键盘按键被捕获
- [ ] AC03: 录制时系统托盘图标变为红色状态，悬浮显示录制时长和事件数
- [ ] AC04: 停止录制后成功保存为 `.itscript` 脚本文件
- [ ] AC05: 选择脚本点击回放，鼠标自动移动到录制的坐标位置并执行点击
- [ ] AC06: 回放时键盘按键被正确模拟，包括组合键
- [ ] AC07: 回放前有 3 秒倒计时
- [ ] AC08: 紧急停止热键（Esc/Ctrl+F12）可随时终止回放

- [ ] AC12: 全局热键 `Ctrl+Alt+R` 在任意应用前台时均可触发开始/停止录制

## 边界与异常

| 场景                 | 触发条件                     | 处理方式                                           |
| -------------------- | ---------------------------- | -------------------------------------------------- |
| UAC 权限弹窗         | 管理员权限弹窗出现           | 钩子无法跨越 UAC 边界，提示用户手动操作后恢复回放  |
| 屏幕分辨率变化       | 回放时分辨率与录制时不同     | 弹出警告，建议用户调整分辨率或手动编辑坐标         |
| 目标窗口未打开       | 回放时预期的应用窗口不存在   | 在悬浮窗提示「未找到目标窗口」，允许跳过或停止     |
| 钩子安装失败         | 杀毒软件拦截全局钩子         | 提示用户将应用加入白名单                           |
| 回放中鼠标被用户移动 | 用户在回放期间有手动操作     | 按当前设计回放会覆盖用户操作；紧急停止热键终止回放 |
| 脚本文件损坏         | JSON 解析失败                | 提示脚本文件损坏，无法加载                         |
| 超长录制             | 事件数超过 100000 条         | 提示数据量过大，建议停止录制并分段保存             |
| 回放中系统休眠/锁屏  | 系统进入休眠或屏幕锁定       | 暂停回放，唤醒/解锁后提示用户恢复                  |
| 连点器频率过高       | CPS 设置超过 100             | 限制最大值为 100，提示已达上限                     |
| 连点器与回放冲突     | 同时启动连点器和回放         | 后启动的操作提示冲突，需先停止前一个               |
| 连点目标窗口关闭     | 固定坐标模式下目标位置无窗口 | 继续执行点击（点击桌面），用户可手动停止           |

## 非功能需求

| 类型       | 要求                                                                         |
| ---------- | ---------------------------------------------------------------------------- |
| **性能**   | 全局钩子额外 CPU 开销 ≤ 3%，事件处理延迟 ≤ 5ms，不影响系统正常使用           |
| **存储**   | 1 分钟录制的脚本文件大小通常 ≤ 200KB（JSON 文本）                            |
| **安全**   | 钩子为只读监听不拦截输入；脚本文件不包含密码等敏感内容（仅键码无法还原密码） |
| **可靠性** | 钩子异常退出时自动卸载，不残留系统级钩子；脚本保存使用原子写入防损坏         |
| **兼容性** | 仅支持 Windows 10+；兼容高 DPI 缩放场景（使用物理像素坐标）                  |
| **持久化** | 脚本文件、用户偏好（回放速度、热键配置）持久化到应用数据目录                 |
