# 需求规格说明书：全局 UI 现代化重构 (Glassmorphism)

---

## 1. 需求背景与目标

**背景**：
当前项目界面采用基础的 Naive UI 默认样式，色调偏淡、容器边界生硬，缺乏现代软件的高级感和空间层级感。为了提升产品的视觉表现力及用户体验，决定进行全站 UI 风格大改版。

**目标**：
基于已验收的设计概念图稿（`prototype_all_views.html`），将 **Glassmorphism（毛玻璃质感）** 设计语言全面应用到整个 Electron 应用中。核心目标包括：

1. 建立沉浸式的空间背景体系与多维度的阴影体系。
2. 彻底移除生硬的组件边框，代之以模糊卡片和大圆角。
3. 规范化操作微交互（悬停弹性、点击沉降）。
4. 实现高质量的深色模式平滑切换体验。

---

## 2. 设计规范说明 (Design Tokens)

新设计的核心基调由一套全新的 CSS Token 驱动。所有 Vue 组件在重构时必须严格遵守并迁移至这套规范。

### 2.1 颜色系统

| Token 名称        | 浅色模式值                  | 深色模式值                  | 作用说明                   |
| :---------------- | :-------------------------- | :-------------------------- | :------------------------- |
| `--bg-body`       | `#f5f7fa`                   | `#0f111a`                   | 最底层容器背景色           |
| `--bg-sidebar`    | `rgba(255, 255, 255, 0.7)`  | `rgba(26, 30, 41, 0.7)`     | 侧边栏及常驻模块毛玻璃底色 |
| `--bg-card`       | `rgba(255, 255, 255, 0.85)` | `rgba(30, 35, 48, 0.6)`     | 主控制卡片、白板底色       |
| `--bg-card-hover` | `rgba(255, 255, 255, 0.95)` | `rgba(30, 35, 48, 0.8)`     | 卡片 Hover 状态加深        |
| `--primary`       | `#5c6bc0`                   | `#5c6bc0`                   | 品牌色/主操作按键色        |
| `--primary-light` | `rgba(92, 107, 192, 0.1)`   | `rgba(92, 107, 192, 0.15)`  | 活跃状态、悬浮高亮底色     |
| `--text-main`     | `#2d3748`                   | `#f7fafc`                   | 主体标题、高亮文本         |

### 2.2 视觉特效与几何特征

- **全局光效**：应用层背景应带有一层渐进的彩色光晕混合，用以映衬玻璃质感。
  `radial-gradient(circle at 15% 50%, rgba(92, 107, 192, 0.08), transparent 25%)`
- **毛玻璃效果 (Backdrop)**：标准材质应用 `backdrop-filter: blur(12px)`。
- **圆角 (Border Radius)**：
  - 微型组件（按钮/状态条）：`12px` (`--radius-md`)
  - 标准卡片容器：`16px` (`--radius-lg`)
  - 大区块包裹容器：`24px` (`--radius-xl`)

### 2.3 微交互 (Micro-interactions)

- **Hover 抬升效果**：功能面片、拖拽区悬停时通过 `transform: translateY(-2px)` 实现悬浮感，并伴随阴影面积增加。
- **按钮动态发光**：Primary Button 默认具备 `0 4px 12px rgba(102, 126, 234, 0.3)` 阴影，Hover 状态扩大模糊半径提升空间感知。
- **转场过渡**：各个子页面之间切换时，统一应用 FadeIn 淡入与向上滑动的位移动画。

### 2.4 人性化操作与体验优化 (UX Best Practices)

为了让系统表现得更加“聪明”贴心，除了视觉更新外，全站的所有页面必须同时落实以下操作体验升级：

1. **大面积宽容触控 (Fitts's Law)**：
   - 所有的可交互元素（按钮、开关、卡片）必须有充足的点击热区（Padding）。
   - 文件拖拽上传区域不应该仅限于中间小图标，背景卡片及留白地带均应触发拖拽热区。
2. **“防呆”与即时反馈 (Error Prevention & Feedback)**：
   - **智能空状态**：列表无文件时，不要仅显示枯燥的文字，配合明确的引导动作（例如“点击这里添加文件”）。
   - **禁用反馈**：在正在转换或压缩过程中，明确在光标（`cursor: not-allowed`）和按钮色条上体现不可用状态，防止用户的重复点击焦虑。
3. **全链路状态护航 (Visibility of System Status)**：
   - 使用带色彩感知的小微章（Pill Badge），即使在快速扫视下（比如绿色代表成功、红色代表失败、律动条代表处理中），用户也能立刻把握进程。
4. **一键快捷覆盖**：
   - 大量设置项（例如输出目录）默认采用“最符合常理的选择（如：与原文件同目录）”，但是需要给进阶用户提供“一键更改”的极简入口，而不遮挡主视线。
5. **记忆与惯性 (System Memory)**：
   - 对部分配置（格式选择、压缩比例等）未来要做到状态缓存记忆，即使页面切换也不丢失当前设定的状态感。
6. **信息降噪与截断 (Progressive Disclosure)**：
   - 对于超长的文件名或输出路径，必须采用“中间截断”或“尾部点点点 (`text-overflow: ellipsis`)”处理，同时在悬停时提供完整信息的 Tooltip 提示，绝不允许长文本破坏原本优雅的卡片布局。
7. **不可逆操作的拦截 (Destructive Action Guards)**：
   - 诸如“清空列表”或“删除特定文件”等销毁类操作，在进行按钮设计时需要采用特定的警示色（如 Danger Red），并在清空整个列表前给予轻量级的二次确认或保留撤销（Undo）机会。
8. **键盘可访问性增强 (Keyboard Navigation)**：
   - （由于是桌面端工具衍生）操作应该顺滑支持全局热键映射以及基础的 Tab 键焦点切换。保证获取焦点的元素有清晰的发光轮廓线（Focus Ring），而非浏览器默认的生硬蓝框。
9. **深浅模式的智能感知与持久化 (Theme Persistence & System Match)**：
   - 不仅要支持手动切换浅色/暗黑模式，系统必须在本地持久化用户选择。同时，默认初始化时应首先尝试跟随操作系统级别的 `prefers-color-scheme`，实现“开箱即用”的最佳夜间视觉。
10. **加载与阻塞反馈 (Skeleton & Loading Overlays)**：
    - 切页或组件挂载时的短暂空白期，应引入轻量级的骨架屏（Skeleton）或全屏高斯模糊遮罩（配合微秒级动画），避免生硬的 DOM 闪烁。
11. **沉浸式交互区域支持 (Immersive Dragging)**：
    - 由于是 Electron 桌面 App，界面头部的空白区域以及侧边栏的顶端必须正确配置 `-webkit-app-region: drag`，保留原生的窗口拖拽体验（注意不能覆盖控制按钮）。

## 3. 功能模块改造清单

本次重构涵盖全站的所有路由页面，以下是每个页面的具体改点与验收标准。

### 3.1 核心框架 (`App.vue`)

- **结构改造**：去除现存 `NLayout` 组件侧栏的硬件分离线条感，调整为漂浮式侧边导航模式。
- **Logo 展示**：替换为设计稿中的双紫蓝渐变文本 `linear-gradient(135deg, #667eea, #764ba2)`。
- **侧栏菜单**：当前项移除背景实色，改为 `--primary-light` 背景，Icon 和字体同时着色 `--primary`，边缘全圆角包裹。
- **深色模式按钮**：剥离原本的绝对定位圆圈按钮，改为贴近底部的长形条段可点击切换。

### 3.2 子视图模块一：SVG 查看器 (`SvgViewer.vue`)

- **顶部工具条**：构建悬浮工具栏 (Floating Action/Search Bar)。搜索框应用深度药丸形状 (Pill shape)。
- **布局切换器**：引入分段控制器 (Segmented Control)，取代传统的下拉菜单，增加点击滑动过渡。
- **卡片方阵**：SVG卡片在背景融合度上应强化 Hover 时的边框染色和比例微放。

### 3.3 子视图模块二：图片压缩 (`ImageCompress.vue`)

- **布局调整**：弃用原来的全宽表格平铺，采用 `左右不对称` 或 `侧滑卡片组` 的布局。
- **交互滑块**：重新实现一个高精度的范围滑块（Range slider），替换原生组件样式。
- **响应区**：新增“拖入空地即捕捉”的全向吸附拖拽交互区域。

### 3.4 子视图模块三：万能连点器 (`ClickerView.vue`)

- **面板卡片化**：将大量表单元素（切换模式、键位热键）重组为两组互对逻辑的“设置组”。
- **单选组改造**：大规模采用圆角的分段选择器 (Segmented Control) 代替 Select 组件或普通 Radio 组件。
- **键位显示组件**：封装 `hotkey-display` 类 Mac 小写键帽风格，具备明确的代码字体与内阴影。

### 3.5 子视图模块四：文档转换与格式转换 (`DocConvertView.vue` / `FormatConvert.vue`)

- **核心控件 (联排下拉框)**：实现两个 `NSelect` 并列、中间融合箭头指向的新颖 UI 组件。
- **拖拽控制中心**：大幅弱化文件上传区的边框，仅保留核心图示。
- **列表态管理**：上传成功的文件队列列表进行紧凑化排版，移除厚重的进度条，改用内置发光小徽标 (Pill Badge) 代表等待/完成/失败状态。

## 4. 技术实施方案建议

1. **样式迁移**：
   重置 `src/styles/variables.css`，直接注入原型的所有 `--` token。通过控制 `<body>` 上附加 `dark-theme` 类来驱动媒体查询和颜色置换。
2. **NaiveUI 主题覆写**：
   因组件深度依赖 `naive-ui`，需全面配置其全局的 `themeOverrides`（替换它的 Primary Color、Border Radius 及内部阴影映射），以便 `NButton`, `NSelect` 等基础控件兼容 Glassmorphism 计算。
3. **分阶段迭代**：
   推荐首先重构 `App.vue`（主体基建），随后按照 **SVG模块 → 压缩模块 → 转换模块 → 连点器** 的顺序逐个击破，期间随时用深暗模式检验可视性。

---

## 5. 验收标准

1. Vue 组件界面效果需与 `prototype_all_views.html` 的美学与空间分配**高度一致（90%以上准度）**。
2. 所有组件及悬浮面板的边缘不得有遗漏的方角像素。
3. 一键切换暗黑模式时，颜色变换瞬间丝滑不生硬，且所有信息层级及对比度（WCGA等级）保持优秀可见性。
