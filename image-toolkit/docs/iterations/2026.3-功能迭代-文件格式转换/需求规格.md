## 需求背景

Universal Toolkit 当前已具备图片压缩和图片格式转换能力，但在日常办公中，用户高频使用的**文档格式转换**需求（如 PPT→Word、Word→Markdown、PDF→Word 等）仍需依赖在线转换工具或付费软件。

本次迭代目标：**新增文档格式转换模块，覆盖 11 种主流文档格式互转方向，提供离线、免费、批量的转换能力，并支持 Markdown/HTML 实时预览。**

## 目标用户

| 用户角色   | 特征                         | 使用场景                               |
| ---------- | ---------------------------- | -------------------------------------- |
| 办公白领   | 日常处理文档，不愿装多个工具 | PPT 转 Word 提取文字、Word 转 PDF 发送 |
| 技术人员   | 需要将文档转为 Markdown 归档 | Word/HTML 转 Markdown，写博客/文档     |
| 学生       | 论文排版、格式兼容           | PDF 转 Word 编辑修改、Word 转 PDF 提交 |
| 内容创作者 | 多平台内容发布               | Markdown 转 HTML/Word，跨平台适配      |

### 模块一：文档格式转换核心（11 种方向）

#### F1.1 Office 文档转换

| 编号   | 转换方向            | 描述                                      | 技术方案                    | 优先级 |
| ------ | ------------------- | ----------------------------------------- | --------------------------- | ------ |
| F1.1.1 | **Word → PDF**      | `.docx` 转 PDF，保留排版和样式            | LibreOffice CLI             | 🔴 高  |
| F1.1.2 | **Word → Markdown** | `.docx` 转 `.md`，提取标题/列表/表格/图片 | `mammoth` + `turndown`      | 🔴 高  |
| F1.1.3 | **Word → HTML**     | `.docx` 转 HTML，保留基本样式             | `mammoth`                   | 🟡 中  |
| F1.1.4 | **PPT → Word**      | `.pptx` 幻灯片内容提取为 Word，每页一节   | `pptx` 解析 + `docx` 生成   | 🔴 高  |
| F1.1.5 | **PPT → PDF**       | `.pptx` 转 PDF（每张幻灯片一页）          | LibreOffice CLI             | 🔴 高  |
| F1.1.6 | **PPT → 图片**      | `.pptx` 每页导出为 PNG/JPG                | LibreOffice → PDF → `sharp` | 🟡 中  |
| F1.1.7 | **PDF → Word**      | PDF 转可编辑 `.docx`（基于文字层提取）    | `pdf-parse` + `docx`        | 🔴 高  |
| F1.1.8 | **PDF → Markdown**  | PDF 转 Markdown                           | `pdf-parse` + 格式化        | 🟡 中  |

### 功能边界

- ✅ 包含：11 种文档格式互转（Word/PPT/PDF/Markdown/HTML）
- ✅ 包含：批量转换、进度显示、输出目录管理
- ✅ 包含：Markdown/HTML 实时预览
- ✅ 包含：大文档支持（上百页，流式处理）
- ❌ 不包含：Excel→CSV（已砍掉）
- ❌ 不包含：OCR 图片识别转文字（PDF 扫描件不支持）
- ❌ 不包含：复杂排版的 100% 还原（如 PDF 复杂表格）
- ❌ 不包含：在线转换 / 云端处理（全部离线本地完成）

## 用户故事

- 作为**办公白领**，我希望**将 PPT 转为 Word**，以便**快速提取幻灯片中的文字内容进行编辑**
- 作为**技术人员**，我希望**将 Word 文档转为 Markdown**，以便**归档到 Git 仓库或发布到博客**
- 作为**学生**，我希望**将 PDF 论文转为 Word**，以便**对文档内容进行编辑修改**
- 作为**内容创作者**，我希望**将 Markdown 转为 Word/HTML**，以便**在不同平台发布适配的格式**
- 作为**普通用户**，我希望**批量转换多个文件并看到进度**，以便**不需要一个一个处理和等待**
- 作为**普通用户**，我希望**转换完的 Markdown 能直接预览**，以便**确认格式正确后再使用**

## 技术方案

| 转换方向        | 推荐方案                        | npm 包                         |
| --------------- | ------------------------------- | ------------------------------ |
| Word → PDF      | LibreOffice CLI (soffice)       | `libreoffice-convert`          |
| Word → Markdown | mammoth.js 解析 docx + turndown | `mammoth`, `turndown`          |
| Word → HTML     | mammoth.js                      | `mammoth`                      |
| PPT → Word      | pptx 解析 + docx 生成           | `pptx-parser`, `docx`          |
| PPT → PDF       | LibreOffice CLI                 | `libreoffice-convert`          |
| PPT → 图片      | LibreOffice → PDF → sharp 渲染  | `libreoffice-convert`, `sharp` |
| PDF → Word      | pdf-parse 提取文字 + docx 生成  | `pdf-parse`, `docx`            |
| PDF → Markdown  | pdf-parse + 格式化              | `pdf-parse`                    |
| Markdown → HTML | marked / markdown-it            | `marked`                       |
| Markdown → Word | marked → HTML → docx            | `marked`, `html-docx-js`       |
| HTML → Markdown | turndown                        | `turndown`                     |

> **外部依赖**：涉及 LibreOffice 的转换（Word→PDF、PPT→PDF、PPT→图片）需要用户安装 LibreOffice 7.x+。应用启动时自动检测并提供安装指引。

## 业务规则

| 编号 | 规则                     | 说明                                                              |
| ---- | ------------------------ | ----------------------------------------------------------------- |
| BR01 | 文件去重基于路径         | 同一路径文件不可重复添加                                          |
| BR02 | 源格式与目标格式不可相同 | 下拉框选择时自动排除当前源格式                                    |
| BR03 | 输出目录路径记忆         | 持久化到本地配置                                                  |
| BR04 | 保留原文件               | 转换后不删除/不覆盖源文件                                         |
| BR05 | 单文件失败不阻断批量     | 失败文件标记错误，继续处理后续文件                                |
| BR06 | LibreOffice 依赖检测     | 涉及 PPT/Word→PDF 转换时检测 soffice，不可用时弹出安装指引        |
| BR07 | 文件大小限制             | 单文件上限 100MB，超出提示                                        |
| BR08 | 图片提取到同名目录       | Word→MD 图片提取到 `文档名_images/`，MD 用相对路径 `./xx_images/` |
| BR09 | 大文档流式处理           | 上百页文档采用分块处理，避免内存溢出                              |

## 边界与异常

| 场景               | 触发条件                | 处理方式                                   |
| ------------------ | ----------------------- | ------------------------------------------ |
| LibreOffice 未安装 | 选择需要 soffice 的转换 | 弹窗提示安装指引，提供下载链接             |
| PDF 为扫描件       | PDF 内无文字层          | 提示「此为扫描件，无法提取文字」           |
| 文件超过 100MB     | 单文件 > 100MB          | 提示「文件过大，请拆分后再转换」           |
| 文件损坏           | 无法读取/解析文件       | 标记失败，显示「文件损坏或格式不支持」     |
| 输出目录不可写     | 权限不足或磁盘已满      | 提示「无法写入输出目录，请检查权限」       |
| PPT 含视频/音频    | 幻灯片嵌入多媒体        | 跳过多媒体，仅提取文字和图片               |
| Word 含宏          | .docm 文件              | 提示「不支持含宏文档，请另存为 .docx」     |
| 大文档内存溢出     | 上百页文档转换          | 采用流式/分块处理，限制并发数              |
| 输出目录已不存在   | 记忆的路径已被删除      | 回退到默认（源文件同目录），提示路径已失效 |
